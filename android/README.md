# TeamMarhaba Android (Capacitor hybrid, direct APK) — TM-278

A **Capacitor** Android host that wraps the hosted TeamMarhaba web UI
(`https://teammarhaba.web.app`), shipped as a **direct-download APK** — no Play Store. It gives phone
users a native-feeling app while the real product stays the single responsive web codebase.

> **TM-278 (root of epic TM-277):** this replaced the hand-rolled TM-231 Kotlin WebView shell with
> Capacitor so native plugins can be added from one codebase. The load model is unchanged — see below.

## Hosted-URL load model (do not change without a ticket)

Capacitor loads the **live hosted site** via `server.url` in `../capacitor.config.json`
(`https://teammarhaba.web.app`) instead of bundling the web build into the APK. Every Firebase
Hosting deploy therefore reaches the installed app **instantly**, with no APK re-release — the same
behaviour the TM-231 shell had. Capacitor adds only the native bridge + plugin layer on top.

`webDir` (`web/src`) is the fallback bundle Capacitor copies into the APK assets; it is **not** what
loads on device while `server.url` is set. (`web/src` is a no-build vanilla-JS SPA, so there is no
`dist/`.)

## WebView env-signal contract (auth)

The shell must signal "I am a WebView" so the web picks **redirect-mode** OAuth (a WebView has no
popup surface) and Google stays hidden on device (TM-275). Honoured by
`web/src/assets/auth-env.js#isWebViewEnv`, which is true if EITHER `window.TEAMMARHABA_WEBVIEW === true`
OR `typeof window.TeamMarhabaWebView !== "undefined"`. `MainActivity.kt` provides **both**, plus a UA
marker:

| Signal | Where |
|---|---|
| `window.TEAMMARHABA_WEBVIEW = true` injected at every page start | `MainActivity.kt` `WebViewListener.onPageStarted` |
| `TeamMarhabaWebView` JS bridge object | `MainActivity.kt` `addJavascriptInterface` + `TeamMarhabaJsBridge.kt` |
| `TeamMarhabaWebView` appended to the User-Agent | `android.appendUserAgent` in `capacitor.config.json` |

First-party + third-party cookies are enabled (`MainActivity.kt`) and Capacitor enables DOM storage
by default, so the Firebase `/__/auth/**` redirect + reCAPTCHA round-trip persist their initial state
(see `docs/agents/webview-auth-contract.md`). Avatar upload (`<input id="profile-avatar-file">`) is
serviced by Capacitor's `BridgeActivity` `onShowFileChooser` natively.

## Build (debug)

Prerequisites: **JDK 21**, **Node 20+**, and the **Android SDK**. Point Gradle at the SDK via
`ANDROID_HOME` or `android/local.properties` (gitignored): `sdk.dir=/path/to/Android/sdk`.

```bash
# from the repo root
npm ci
npx cap sync android
cd android
./gradlew assembleDebug
# → app/build/outputs/apk/debug/app-debug.apk  (applicationId app.teammarhaba.webview.debug)
```

Debug builds are signed with the auto-generated Android debug keystore — no secrets, installable for
testing (`./gradlew installDebug`).

> CI (`.github/workflows/android-release.yml`, `compile-check`) does exactly this: `npm ci` →
> `cap sync android` → `assembleDebug`, and uploads the APK artifact. Touched by TM-287 for the
> on-gate build wiring.

## Signing & release

The release signing config in `app/build.gradle` reads the keystore + passwords from Gradle
properties / env — **never committed**. Supply them via `~/.gradle/gradle.properties`, `-P` flags, or
`ORG_GRADLE_PROJECT_*` env (CI), keyed `teammarhaba.releaseStoreFile` / `…StorePassword` /
`…KeyAlias` / `…KeyPassword`. A local `android/app/keystore.properties` (gitignored) is also honoured.
If the props are absent, `assembleRelease` still builds but produces an **unsigned** APK.

The release CI job (`android-release.yml`, `release`, on a `v*` tag or manual dispatch) decodes the
keystore from the TM-245 secrets, stamps `versionName` to the live web `buildVersion` via
`-PtmVersionName`/`-PtmVersionCode`, builds the **signed** APK, and publishes it +the `/download`
page to Firebase Hosting. It **fails loud** if the signing secrets are absent.

> ⚠ **Keystore custody is critical.** Losing the release keystore (or its passwords) means you can
> never ship an update the installed app accepts. Back it up in a secret vault, never the repo —
> human ticket **TM-245**.
